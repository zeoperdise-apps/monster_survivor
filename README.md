# Monster Survivor - プロジェクト仕様書

## 概要
Monster Survivorは、プレイヤーがモンスターと戦いながらレベルアップしていくアクションゲームです。プレイヤーは画面中央に固定され、周囲の敵を倒して経験値を得てレベルアップし、より強力な武器やアクセサリを獲得できます。

## ゲーム要素

### プレイヤー
- 位置: 画面中央に固定（カメラがプレイヤーに追従、フィールドの広さは無制限）
- 移動: WASDまたは矢印キーで移動
- サイズ: 半径24（スケルトンと同じ大きさ）
- HP: 100（最大値。アクセサリ効果で加算される）
- レベル: 初期値1
- 経験値(XP): 初期値0、次レベルまでの必要量は初期50で、レベルアップごとに1.2倍ずつ増加
- 向き: 左右移動キーの入力方向を向く（攻撃方向・画像の左右反転に使用）
- 衝突判定: 敵と接触するとダメージ（5、防御力ボーナスで軽減）を受ける。被弾後0.5秒（30フレーム）は無敵時間があり連続ヒットしない
- HPが0になるとゲームオーバー（生存時間を表示しリスタート可能）

### 武器（js/weapons.js）
- 開始装備: 鞭（Whip）1本のみ、レベル1
- 装備可能な武器は最大4本まで
- 全10種類がレベル1から選択肢に登場する（レベル制限なし）
- 装備している武器はすべて（最大4本）タイプ別のロジックで自動的に攻撃する
  - 近接（鞭・剣・斧・槍・雷のハンマー・毒の短剣）: プレイヤーの向いている方向・射程内の敵に直接ダメージ
  - 遠距離（弓・氷の弓）: 射程内の最も近い敵に単体狙いの弾を発射
  - 魔法（杖・火の杖）: 射程内の最も近い敵に弾を発射し、着弾地点の周囲（半径60）にも範囲ダメージ

### 敵（js/enemy_types.js）
- 生成: 100フレームごとに、プレイヤーの現在位置を基準とした画面端付近のランダムな位置から生成
- 移動: プレイヤーの方向に向かって移動
- プレイヤーレベルに応じて出現する敵種類が拡張される

### アイテム
- XP宝石（緑色）
  - 経験値: 10
  - 敵を倒すとその場にドロップ、プレイヤーが近づくと自動収集

### アクセサリ（js/accessories.js）
- ボーナスは100%を基準とした％表記（例: 110%なら+10%）。同じアクセサリを複数レベル獲得すると加算される
- ゲーム開始時から以下4種が自動的に装備・適用される
  - 力の腕輪: 攻撃力+10%
  - 神速の靴: 移動速度+10%
  - 生命の首飾り: 最大HP+20%
  - 疾風の手甲: 射撃速度+15%
- 守護の護符（防御力+10%、被ダメージ軽減）はレベルアップ時の選択肢として入手可能（初期装備には含まれない）
- レベルアップ時の選択肢には、未所持のアクセサリが1つ登場する（全種類所持済みの場合は代わりに武器の選択肢が入る）

## ゲーム機能

### レベルアップ
- XPが次のレベルに必要なXPを達成するとレベルアップし、XPは0にリセットされる
- レベルアップ時、以下から3択（+スキップ）を提示
  - 武器の新規入手 or レベルアップ（最大2種類分）
  - ステータス強化アクセサリ（1種類、レベル条件あり）
  - 10%の確率で全武器共通のランダム強化（攻撃力+10%／射撃速度+15%／移動速度+10%／回復＆最大HP+20）

### ユーザーインターフェース
- レベル表示
- 経験値表示（現在値/必要値）
- HP表示
- ゲーム時間表示
- 装備武器アイコン＋レベル表示
- ステータスボーナス表示（攻撃力・移動速度・射撃速度・最大HP・防御力、いずれも100%基準の％表記）

## 技術仕様

### プログラミング言語
- HTML5 Canvas + JavaScript

### コンポーネント構成
1. index.html: ゲームの基本構造とUI要素
2. js/main.js: ゲームループ、プレイヤー、敵生成、戦闘、UI更新など
3. js/weapons.js: 武器データと武器の取得・レベルアップ処理
4. js/enemy_types.js: 敵データとレベル別の出現制御
5. js/accessories.js: アクセサリのデータ定義と付与・ボーナス計算処理
6. css/style.css: ゲームUIのスタイル
7. img/*: 画像データ

### ゲームループ
- 60FPSでゲームが実行される
- canvas要素を使用して描画
- カメラはプレイヤーを常に中央に追従させる

## 開発方針
- 現在の仕様を維持しつつ、機能追加や改善を行う際にはこの仕様書も更新する。
- プレイヤーがゲームに集中できるシンプルなUIを維持する。
